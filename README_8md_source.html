3 # Prior Hessian
4 Library for fast computation of log-likelihoods, and derivatives, of multivariate priors defined as composites of univariate distributions and multivariate distributions especially
5 Archimedean copulas.

34 ## Motivation
35 
36 For many likelihood-based methods, they can be extended to Bayesian methods like MAP Estimation and MCMC Posterior sampling,
37 by incorporating a prior. This prior must provide fast methods for computing log-likelihood and it's derivatives over the
38 parameter space. The prior log-likelihood, as well as it's gradient and hessian are then added to the equivalent quantities from the likelihood
39 to create a Bayesian objective for MAP Estimation.

41 ## Static Polymorphism
42 
43 The PriorHessian library is designed using static polymorphism (templates), and as such avoids virtual functions for small-grained tasks, and instead uses templates, which allow many small functions to be inlined. This aggressive inlining by the compiler produces log-likelihood, gradient, and hessian functions that are nearly as fast as hand-coded functions. But our flexible [`CompositeDist`]() class is able to be easily created with any mix of [`UnivariateDist`]() and [`MultivariateDist`]() elements.
44 
45 In [Mappel](https://github.com/markjolah/Mappel) we use this ability to create heterogeneous priors for each Model's parameters, (e.g., [x,y,I,bg,sigma]).
46 
47 Functionally, the PriorHessian library stores sequences of distributions as `std::tuples`. Using this approach as opposed to
48 the runtime polymorphism of using `std::vector<std::unique_ptr<Base>>` gains several advantages.
49 Most importantly, without the need for virtual functions, the tuple-based approach has the ability to inline the many
50 small computational functions that must be combined for every call to compute the log-likelihood or other computationally important quantities.

52 ## Computations available
53 
54  * `cdf` - cumulative distribution function
55  * `pdf` - probability density function
56  * `llh` - log-likelihood (log of pdf)
57  * `rllh` - relative log-likelihood (log of pdf without constant terms)
58  * `grad` - derivative of log-likelihood (or equivalently of relative-llh)
59  * `grad2` - 2nd-derivative of log-likelihood
